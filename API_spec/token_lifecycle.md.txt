(A) Four stages for the Secure System Design documentation:

1.Generation: Tokens are generated using the HS256 (HMAC with SHA-256) algorithm once a user is authenticated. The payload includes user_id, role, and exp (expiration).


2.Usage: The client app must include the token in the Authorization: Bearer <token> header for all /api/alerts requests.


3.Expiration: Access tokens will expire every 15 minutes to minimize risk if a token is intercepted.


4.Revocation: In case of a suspected cyber threat or credential leak, the alert_id or user_id can be added to a "blacklist" in the secure storage to immediately invalidate the token.
