## AI-ML Feature Engineering Pipeline (AI004)


## 1. Overview

This document describes the feature set generated by the AI004 Feature Engineering pipeline. The pipeline transforms raw multi-domain datasets (cyber, hazard, temporal, and geospatial) into structured, ML-ready features for downstream analytics such as risk prediction, anomaly detection, and event correlation modeling.



## 2. Hazard Features

These features quantify disaster severity, event impact, and short-term hazard dynamics.

* disaster_severity_score → weighted severity representation
* event_intensity_index → severity × duration_hours
* hazard_normalized → normalized severity scale
* severity_change_rate → temporal change in severity
* hazard_trend_index → rolling mean severity trend
* severity_volatility → short-term variability in severity
* multi_event_overlap_flag → indicator of extreme event conditions



## 3. Cyber Features

These features capture cyber incident activity, intensity, and behavioral shifts.

* cyber_incident_count → total incident volume
* cyber_intensity_score → normalized cyber activity level
* scam_spike_rate → sudden changes in incident frequency
* cyber_attack_frequency → rolling event occurrence count
* cyber_growth_rate → percentage change over time
* cyber_volatility → instability in cyber activity
* incident_peak_flag → above-average activity indicator



## 4. Temporal Features

These features model time-based dependencies and sequential behavior.

* rolling_cyber_mean → moving average of cyber incidents
* time_since_last_event → temporal gap between events
* ema → exponential moving average of incidents
* lag_1 → previous timestep value
* lag_2 → two-step lag value
* time_decay_factor → exponential time-weighting function



## 5. Geo Features

These features represent spatial and regional risk distribution.

* geo_risk_zone_score → severity-based regional risk score
* location_encoded → encoded categorical location feature
* regional_event_count → event density per location



## 6. Risk Features

These features combine cross-domain signals to estimate overall system risk.

* combined_risk_index → interaction between severity and cyber activity
* adaptive_risk_index → normalized risk adjusted by recent trends



## 7. Anomaly Features

These features detect statistical deviations and abnormal behavior.

* z_score → statistical deviation from mean activity
* outlier_flag → binary anomaly detection indicator



## 8. Summary

The pipeline converts heterogeneous raw data into structured, machine-learning-ready feature representations. These features support:

* Risk prediction modeling
* Cyber-hazard correlation analysis
* Temporal anomaly detection
* Regional risk assessment systems
