# Key Asset Identification
Key assets in the PHOENIX system are identified based on analysis of the cleaned threat dataset, which includes phishing, scams, ransomware, data breaches, misinformation, and system attacks.
---
## 1. Data Assets
### 1.1 Personal and Victim Data
Sensitive information such as names, phone numbers, emails, and health records, which are targeted in phishing and data breach incidents.

### 1.2 Financial Information
Bank details and payment information targeted in scams and fraudulent donation attacks.

### 1.3 System Credentials
Usernames, passwords, and authentication data targeted through phishing and malware attacks.

### 1.4 Alert Messages
Official emergency alerts generated by the system, which must remain accurate and protected from tampering or spoofing.
---

## 2. System Components

### 2.1 ADCRS System
AI-based system responsible for analysing threat data and detecting cyber risks.

### 2.2 TEAVS System
Alert generation and verification system, critical due to risks of fake alerts identified in the dataset.

### 2.3 API Interfaces
System entry points that can be exploited in API abuse attacks.
---

## 3. Security Assets

### 3.1 Cryptographic Keys
Used for digital signatures to ensure alert authenticity and prevent spoofing.

### 3.2 Authentication Tokens
Used to manage secure access and prevent unauthorised system usage.

### 3.3 Access Control Mechanisms
Role-based access controls that restrict system actions.
---

## 4. Storage Assets

### 4.1 Threat Database
Stores collected and analysed cyber threat data.

### 4.2 Alert Logs
Records system activity and alerts for monitoring and auditing.

### 4.3 Key Storage
Secure storage for cryptographic keys.
---

## 5. Operational Assets

### 5.1 System Availability
Critical to ensure system remains functional during disasters and resistant to DDoS or ransomware attacks.

### 5.2 Communication Channels
Channels used to deliver alerts, which must remain secure to prevent misinformation and fake alerts.
---

## Conclusion

The identified assets are derived from real-world cyber threats observed in the dataset, ensuring that PHOENIX focuses on protecting critical data, system components, and communication infrastructure from attacks such as phishing, scams, ransomware, and misinformation.
