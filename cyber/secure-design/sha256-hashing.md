# SHA-256 Hashing

## Overview
In the TEAVS authentication system, user passwords must be securely stored and verified. Using the SHA-256 cryptographic hashing algorithm, passwords are converted into irreversible hash values by the system. This prevents sensitive credentials from being compromised if the database is leaked. 

## Password Security
The TEAVS system must not store passwords in plaintext format. On creation or updation of a password, SHA-256 hashing is applied before storing the password in the database. The database only contains the hashed values. 
When the user logs in, the entered password is
hashed and compared against the previously stored value in the database. 
If the hashes match, authentication is successful. 

## SHA-256 Hashing Process
Steps taken by the system to verify the password:
1. User fills out the login form on Frontend, entering username and password.
2. HTTPS protocol carries the password to Backend.
3. Backend applies the SHA-256 Hashing to the password.
4. The hashed password generated by Backend is compared to the stored hashed password in the Database. 
5. Hashes match - Authentication Successful. 
     Hashes do not match - Access Denied

## Security Benefits
1. Passwords that are stored in the database are not in plaintext format. This ensures that the passwords remain protected in the event of a database compromise. 
2. SHA-256 hashing produces a unique, irreversible hash value, therefore, original readable passwords are not retrievable from the hash. 
3. SHA-256 hashing ensures integrity during authentication by constant verification of user credentials.
